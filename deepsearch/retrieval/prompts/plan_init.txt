## Output Schema
Return a single JSON object with these keys:

- "subqueries": Array of objects, each with:
  - "subquery_id": string like "Q1", "Q2", …
  - "index": array of index names (pick the BEST 1–2 indexes from the allowed list)
  - "q": the natural-language search string for that subquery
  - "dialogue": "true" if the subquery targets spoken words (transcript/topic), else "false"

- "join_plan": How to combine subquery results.
  - "op": "AND" or "OR"
  - "subqueries": list of subquery_ids to join

- "metadata_filters": Object where keys are facet names (from the allowed list) and values are arrays of strings. Only include facets the query clearly implies. Use exact values from the known metadata when possible.

- "preferences": Object of ranking hints (e.g. {"shot_type": ["close-up"]} means prefer close-ups). Optional.

- "fallback_order": Array of facet names in the order they should be dropped if results are empty. Default: {{registry_fallback_order}}

## Guidelines
1. Prefer fewer subqueries (1–3) unless the query has genuinely independent aspects.
2. Pick the most specific index for each concept. If in doubt, use scene_description.
3. For dialogue/lyrics queries, always include "transcript" or "topic" in the index and set dialogue to "true".
4. Do NOT invent facet names outside the allowed list.
5. Do NOT include "person_names", "actors", or "characters" in metadata_filters — these are not supported.
6. The "objects" facet should contain object labels exactly as they appear in the known metadata.
